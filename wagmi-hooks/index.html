1.wagmi && viem基本介绍

viem是ethers.js的替代品，相比后者，viem具有更小的软件包和更佳的性能，由于超高的性能，目前已经被各种头部项目所采用，相信未来选择viem的项目也会越来越多。但是如果论API的完整性的话，还是ethers会更加丰富一些，所以实际使用过程中，有的项目可能会将两者结合起来使用

wagmi是We are gonna make it的缩写，它是一种与以太坊配合使用的React Hooks和实用程序。wagmi提供了大量易于使用的 API，使区块链交互变得更容易。并且它使用React hooks可以简单地组合复杂的功能，我们也可以根据具体需求定制各种不同的解决方案。事实上wagmi其实就是对 viem 的 hooks 再封装，可以看到它的peerDependency就已经写明了依赖

  "@tanstack/react-query": ">=5.0.0",
  "react": ">=18",
  "typescript": ">=5.0.4",
  "viem": "2.x"
},

2.wagmi 所有hooks

wagmi官方的Core模块已经罗列出了所有核心方法，比如getAccount、getBalance、这些方法大多是直接基于viem封装的，可以直接拿来使用，或者二次封装成hooks，这里就不过多介绍。这里主要介绍它封装的hooks函数，这些hooks函数都支持config参数，比如：

1

const account = useAccount({ config })

config参数默认不传时代表wagmi.ts配置，如果传了就代表本次查询是基于一个自定义的配置

2.1.useAccount

useAccount获取当前钱包账户的基本信息，其返回值包含钱包地址address、使用的区块链信息chain、chainId、连接的钱包信息connector、连接状态isConnecting / isReconnecting / isConnected / isDisconnected

还有一个status，它有4个状态'connecting' | 'reconnecting' | 'connected' | 'disconnected'

'connecting'尝试连接到钱包

'reconnecting'尝试重新建立与一个或多个钱包的连接

'connected'至少有一个钱包已连接

'disconnected'未连接至任何钱包

1

const { address, chain, connector } = useAccount()

2.3.useBalance

useBalance获取某个账户、某个链下的余额，入参：

address：这是一个必填项，指的钱包地址。获取该钱包的余额

blockNumber：该区块链下的某个blockNumber。获取处于blockNumber时的钱包余额

blockTag：指定一个blockTag，它有以下分类值'latest' | 'earliest' | 'pending' | 'safe' | 'finalized' | undefined。比如latest，获取最新区块的钱包余额

chainId：指定获取哪个链下的钱包余额

scopeKey：用作缓存的key，如果设置了，并且缓存生效，那么就不会去从链上拿，直到缓存更新。其内部是基于@tanstack/react-query去做的处理，也是基于它实现的缓存

token：指定某个ERC20代币的合约地址，获取该代币的余额

unit：查询结果的单位，'ether' | 'gwei' | 'wei' | number | undefined

query：自定义tanstack query的参数

1
2
3

  const result = useBalance({
    address: '0x4557B18E779944BFE9d78A672452331C186a9f48',
  })

返回结果：

data：包含了此次查询余额的结果，{ decimals: number; formatted: string; symbol: string; value: bigint; }

其余返回结果可以查看文档，有的也是tanstack query的定义

2.8.useCall

useCall 直接去执行某个操作，这个钩子函数有点类似于合约的call函数

nonce：此次操作提供的nonce，代表着交易的标识

value：转账的金额

2.13.useConnect

useConnect连接钱包的钩子函数

入参：

mutation：其实也是tankstack query的入参，包含了gcTime、onMutate、onSuccess等等参数

1

const { connect } = useConnect()

返回结果：

connectors：所有配置或者创建好的connectors，通俗点讲就是钱包的集合

connect：连接钱包的方法，调用后，就可以发起连接钱包账户的操作，它可以配置几个入参variables、onSuccess、onError、onSettled

variables为连接钱包时的一些自定义参数，包括chainId、connectors，onSuccess && onError连接成功或者失败的回调函数，onSettled当连接成功或者突然失败会触发，如果发起多个钱包的连接的话，上面onSuccess的回调会在最后一个连接成功时触发

connectAsync：与connect类似，不同的地方在于它返回一个Promise让我们能去做一些状态控制

data：连接成功后，返回连接的钱包地址信息以及chainId

2.14.useConnections

useConnections返回当前已连接的钱包账户list

1

一个钱包list，每个元素包含了accounts、chainId、connector属性

2.15.useConnectorClient

useConnectorClient 返回一个viem实例，与useClient不同的是，它返回的结果还包含当前已连接钱包的实例数据。所以为了准确获取connector client，会优先选择useConnectorClient

入参：

account：钱包地址，查询包含该钱包的viem实例，如果未找到，则返回为空

chainId：与上面类似，查询该链的viem实例，这个与当前连接的区块链有关，如果未找到，则返回为空

connector：钱包配置数据，默认为当前钱包

query：tankstack query配置

1

const result = useConnectorClient()

返回结果：

data：与useClient不同，useConnectorClient返回的viem实例数据放在了data里面，useClient直接返回的结果直接是一个实例数据

2.19.useEnsAddress

useEnsAddress 通过ENS名去获取对应的钱包地址

data：默认为undefined，当解析成功时返回一个钱包地址

usePrepareTransactionRequest 模拟交易请求，返回一个TransactionRequest类型值

入参：

accessList：请求时，通过指定已访问 的address、存储插槽storageKeys，达到节省gas的目的，详见说明[eip-2930](

chainId：指定查询某个链上的交易

connector：指定使用哪个钱包配置，如果此时未连接到此钱包，那么会有一个默认的账户

data：合约中的calldata

gasPrice：每 gas 支付的价格(单位为 wei)，仅适用于传统交易

maxFeePerGas：每 gas最高费用(单位：wei)，含maxPriorityFeePerGas，仅适用于eip-1559 交易

maxPriorityFeePerGas：每 gas 最高优先费用(单位：wei)，仅适用于eip-1559 交易。主要是提供给矿工的奖励，也叫小费

nonce：交易的标识以及顺序

value：转账金额

parameters：("fees" | "gas" | "nonce" | "type")[] | undefined，额外的查询参数，比如传递 parameters: ['gas', 'nonce']

那么只会提供gas、nonce的准备

poll：轮询的方式，默认false以WebSocket的方式查询，true为非WebSocket方式

pollingInterval：轮询的间隔，毫秒为单位

syncConnectedChain：当已连接的链发生改变时，设置监听器，没太看懂？
