前言

Signer是以太坊账户的抽象，可以用来签名消息和交易，如将签名的交易发送到以太坊网络以执行状态更改的操作。Signer是抽象化的，不能直接实例化，要用一个具体的子类，比如Wallet、VoidSigner、JsonRpcSigner，也就是说，实际使用Signer时用的就是Wallet、VoidSigner、JsonRpcSigner其中一个类的实例

注：本文章基于ethers.js v6

1.Signer的抽象方法

Signer的子类(Wallet、VoidSigner、JsonRpcSigner等示例)可以调用一些公共方法

1.1.connect

signer.connect( provider ) ⇒ Signer

该方法返回一个Signer。比如，我们初始化了一个Wallet实例，然后connect传入一个provider，这个provider可能是JsonRpcProvider、BrowserProvider等，返回的结果Signer就可以调用一些公共方法了，比如发起交易sendTransaction

1.4.call

signer.call(tx: TransactionRequest)⇒ Promise< string >

根据当前区块链状态运行交易来评估交易。这无法改变状态，并且不会产生任何以太币成本，因为它实际上是在模拟执行

1.5.getNonce

signer.getNonce(blockTag?: BlockTag)⇒ Promise< number >

获取指定区块下一次交易的nonce值

1.7.populateTransaction

signer.populateTransaction(tx: TransactionRequest)⇒ Promise< TransactionLike  < string > >

与populateCall有些类似，不过它是准备一个完整的transaction对象。它会填充nonce、from、gasLimit、chainId等等信息，在调用之前，请注意钱包余额是否足够，否则会报错

1.10.signMessage

signer.signMessage(message: string | Uint8Array)⇒ Promise< string >

使用钱包签名一个消息，返回的是65字节的签名结果，参考以太坊中数字签名、消息的概念

1.11.signTransaction

signer.signTransaction(tx: TransactionRequest)⇒ Promise< string >

返回一个已签名的交易的Promise

1.12.signTypedData

signer.signTypedData(domain: TypedDataDomain, types: Record< string, Array< TypedDataField  > >, value: Record< string, any >)⇒ Promise< string >

同样是签名一个消息，不过是eip-712提议格式的，参考以太坊中数字签名、消息的概念

3.JsonRpcSigner

JsonRpcSigner是一个简单的Signer，它由一个连接的JsonRpcProvider支持。与wallet不同的是，通过钱包工具与链上连接，获一个JsonRpcSigner后，就可以对链上进行读写

import { BrowserProvider, JsonRpcSigner } from 'ethers'
import { getConnectorClient } from '@wagmi/core'

async function getEthersSigner({ chainId }: { chainId?: number } = {}) {
  // 获取client信息，传给clientToSigner
  const client = await getConnectorClient(config, { chainId })
  return client ? clientToSigner(client) : undefined
}

function clientToSigner(client: Client<Transport, Chain, Account>) {
  const { account, chain, transport } = client
  const network = {
    chainId: chain.id,
    name: chain.name,
    ensAddress: chain.contracts?.ensRegistry?.address,
  }
  
  // transport为https://docs.ethers.org/v6/api/providers/#Eip1193Provider
  // network 为连接的链的信息
  // new JsonRpcSigner传递 new BrowserProvider、address
  const provider = new BrowserProvider(transport, network)
  const signer = new JsonRpcSigner(provider, account.address)
  return signer
}

const signer = new JsonRpcSigner(provider, account.address)，signer就可以拿去初始化合约实例，实现与以太坊链上的交互

4.VoidSigner

new VoidSigner(address: string, provider?: null | Provider)

VoidSigner是一个简单的Signer，它不能签名，当API需要signer作为参数时，它作为只读的signer是有用的，但它只能携带只读的操作。比如，初始化合约实例，作为参数传递，查询链上信息

// 获取此帐户的token数量
